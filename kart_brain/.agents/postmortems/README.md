# Postmortems

This directory holds detailed analysis of significant failures. Use when an error is complex enough that the one-line entry in `error_log.md` isn't sufficient.

## Format

```markdown
# YYYY-MM-DD - Title

## What happened
Detailed description of the failure, including timeline.

## Root cause
Why it happened — the underlying technical reason.

## Impact
What was lost (time, data, broken state).

## Fix applied
Exact changes made to resolve the issue.

## Prevention
What systemic changes were made to prevent recurrence.

## Lessons learned
Higher-level takeaways.
```
