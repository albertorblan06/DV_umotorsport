## Goal 1: Fix Kart Oversteering with YOLO Pipeline

Context: `steering_gain` in `cone_follower_node.py` defaulted to `1.0`.
On moderate lateral cone offsets the gain multiplied `atan2(lateral, forward)`
past `max_steer=0.5`, causing saturation and oscillation through turns.

**Status: COMPLETE (2026-03-11)**

### Steps

- [x] 1. **Locate Parameter:** `steering_gain` is declared in
  `src/kart_sim/scripts/cone_follower_node.py` (line 44) and applied at line 219:
  `steer = clamp(steering_gain * atan2(mid_l, mid_f), -max_steer, max_steer)`.
- [x] 2. **Write Diagnostic Test:** Created `tests/test_steering_gain.py` with four
  unit tests (no ROS needed). Confirmed that `gain=1.0` saturates on
  tight-corner cone geometry (`atan2(1.5, 2.0) = 0.64 rad > max_steer=0.5`)
  while `gain=0.5` stays proportional and halves the output.
- [x] 3. **Apply Fix:** Changed default to `0.5` by passing it as a parameter in
  `src/kart_bringup/launch/autonomous.launch.py`. Added `DeclareLaunchArgument`
  so the gain can be overridden at launch time without recompile:
  `ros2 launch kart_bringup autonomous.launch.py steering_gain:=0.4`
- [x] 4. **Committed:** `kart_brain` commit `0ed125b`.

### Notes

- `cone_follower_node.py`'s node default remains `1.0` (untouched) so that
  simulation runs with `simulation.launch.py` are not affected.
- `max_steer` is still `0.5 rad` (~28.6 deg) on both the follower and the bridge.
